# Documentation Policy

## 基本方針

- 永続的に参照する仕様整理は `docs/` 直下の文書に集約する。
- 恒久文書は追記で増やすのではなく、同じファイルを上書き更新して管理する。
- 一時的な検討過程、試算、比較、作業メモは `docs/reports/` に残す。
- `overview.md`は人間のみが編集するプロセス設計の進捗ファイルである。

## 反応器文書

- 反応器の恒久文書は `docs/reactor.md` とする。
- 反応器の現状仕様、入出力、出力の意味は `docs/reactor.md` にまとめる。
- 反応器まわりの実装や理解が更新されたときは、`docs/reactor.md` を上書き更新する。

## `docs/reports/` の使い方

- `docs/reports/` は PR 単位の作業記録に使う。
- 同一 PR で同じ主題の記録が増えても、原則として 1 ファイルにまとめる。
- 恒久文書に残すべき内容は、最終的に `docs/` 直下の文書へ反映する。
