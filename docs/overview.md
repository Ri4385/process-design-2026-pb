### 6.1. 反応器

\- 反応器温度(540 \- 640度 17529の化学商品みたいなやつより、まあ文献によって値は結構違う、減圧しないと多分温度上げれない気がする)  
\- 大きさ(内径、長さ、あと今のコードだと入口体積流量と長さだけ決めてる、自由度的にそこから内径が決まる)  
\- 多段かそうでないかとか(商用プロセスでは多段が一般的で2段が多いっぽい？)  
\- pressure(常圧でよいか？本来は低圧で行うのが有利らしい、2molへ増える反応なので)  
コンテスト資料：商用プロセスでは 200 mmHg 程度\~大気圧より少し高い圧力まで

contest資料より  
\>SM製造プロセスの設計とは、この反応器間再加熱を伴う反応器システムにおいて、反応器システム入口のスティーム/EB比率、反応器入口温度、再加熱方式、反応器数、反応器操作圧力、反応器サイズを主な決定変数として、製品仕様、生産量を満たすのに必要なプロセス構造、プロセス変数を設計する問題となる。

\>過熱器で適当な温度まで昇温した後、反応器システムに供給するために 590\~650°Cまで加熱する必要があるが、加熱炉による間接加熱を行うと、チューブ表面が 1500°Cを超える高温となり、チューブ表面で EB の脱水素反応に加え重合反応、分解反応が起こり、炭素析出の危険性が増大する。そのため、過熱器出口の EB-スティーム流体に、高温(700°C以上)希釈スティーム(流体(5))を加えることで、規定の温度(590\~650°C)まで昇温する。高温希釈スティームの投入量は、反応器システム入口のスティーム/EB モル比率をいくつにするかによって決まり、高温希釈スティームの温度は、同じく反応器システム入口の操作設計温度を何度にするかによって規定される。

探索するパラメータ

| パラメータ | 扱い | 妥当な範囲 | 備考 |
| :---- | :---- | ----- | :---- |
| 各段入口温度 | 最適化変数 | 590から650 ℃ | コンテスト条件を優先するなら590から650 ℃の範囲。過去レポ7の545から606 ℃は現行課題条件より低め。 |
| 圧力 | 最適化変数 | 出口圧力 200mmHg abs以上、入口圧力は圧損込みで決定 | 反応器出口を下限制約に置き、入口圧力を振るのが自然。常圧付近までを候補にできる。 |
| Steam/EB比 | 最適化変数 | 5から8、基準値5 | 過去レポは5を採用している。低すぎるとコーク・分圧面で不利、高すぎると加熱・冷却負荷が増える。 2008コンテストでは触媒への炭素析出を防ぐため5倍以上とあった。 |
| 段数 | 最適化変数 | 2または3段 | コンテスト・商用文脈では2から3段が中心です。1段は比較用ベースライン。 |
| 段長または体積 | 最適化変数 | 目標SM量を満たす範囲。各段L/Dは概ね1から3程度を目安 | 体積そのものは探索で決め、実機寸法は後で並列数と直径から調整するのがよい。 |
| 段ごとの体積配分 | 最適化変数 | 均等配分から、各段20から60 %程度 | 総体積を固定して配分を振ると、温度低下と反応進行のバランスを調整できます。 |
| EB入口流量 | 目標SM生産量に合わせる調整変数 | 目標SM生成量を満たすように自動調整 | 独立した最適化変数ではない。条件ごとに必要Feedが変わる。 |
| EB単通転化率 | 結果指標 | 30から70 %程度、現行値は約54.2 % | 高ければよいとは限らない。副反応、反応器体積、リサイクル負荷とのトレードオフ。 |
| SM選択率 | 結果指標 | 高いほど望ましい。現行値は約93.4 % | 原料原単位に直結する。温度を上げすぎると悪化しやすい。 |
| 未反応EB量 | 結果指標 | 小さいほど分離・リサイクル負荷は小さい | ただし無理に小さくすると副反応や必要体積が増える可能性がある。 |
| 空塔速度 | 実現性制約またはペナルティ | 1から3 m/s程度 | 過去レポ7の設定。圧損、反応器径、滞留時間に関係。 |
| 圧損 | 実現性制約またはペナルティ | 0.2 bar/反応器を基準 | コンテスト指定値です。3段なら反応器のみで0.6 bar。間接再加熱器を入れるならさらに加算する。 流量が多いため、反応器の途中で加圧するのは、コストの観点から非現実的(多分)。 |
| L/D | 実現性制約またはペナルティ | 1から3程度、目安は2 | 単一基で外れる場合は並列化で調整できる。 |
| 反応器直径 | 実現性設計値 | コンテストでは3.35 m | 最適化変数というより、体積から高さを出すための機器設計条件。 |

## 7\. ヒートインテグレーション

これ熱交換でユーティリティコストが結構下がるなら、最適化においてそれも考慮に入れるべきなのでは。

レポート7より  
\>ヒートインテグレーションによりユーティリティーコストは 80.28 億円/年から 19.46 億

GPT comment follows:

妥当です。むしろかなり重要な指摘です。

ヒートインテグレーションでユーティリティコストが \`80.28 億円/年 → 19.46 億円/年\` まで下がるなら、最適化後に後付けで考えるだけでは不十分です。反応器条件を変えると、反応器出口温度、段間再加熱負荷、冷却負荷、蒸留リボイラー負荷、コンデンサー負荷が変わるため、熱回収可能量も変わります。

つまり、本来の目的関数は、「反応器単体の加熱負荷」ではなく、「外部ユーティリティとして最終的に必要な加熱・冷却負荷」で評価すべきです。ただし、最初から完全なヒートインテグレーション込み最適化をやると複雑です。現実的には、次の2段階がよいです。

\*\*1. 反応器候補を複数出す\*\*

温度、圧力、S/EB比、段数、段長を振って、上位候補を数個出します。この時点では簡易ユーティリティ評価でよいです。

\*\*2. 各候補に対してヒートインテグレーション後の外部ユーティリティを再評価する\*\*

各候補について、

\- 高温側ストリーム  
\- 低温側ストリーム  
\- 供給温度  
\- 目標温度  
\- 熱負荷

を整理し、ピンチ解析または簡易熱回収モデルで、外部加熱・外部冷却を見積もります。

そのうえで、

\- HIなし最適案  
\- HIあり最適案

を比較します。

この比較はレポートとしてかなり意味があります。特に、次のような結論が出せる可能性があります。

| 比較結果 | 解釈 |  
|---|---|  
| 最適条件が変わらない | 反応器最適化はヒートインテグレーションに対して頑健です。 |  
| 最適条件が変わる | 熱回収可能性を無視した最適化は局所的で、全体最適ではありません。 |  
| 低温出口案が有利になる | 冷却負荷は下がるが、熱回収源としての価値も下がる可能性があります。 |  
| 高温出口案が有利になる | 反応器出口熱を原料予熱やリボイラーに使えるため、単純な反応器負荷評価より有利になる可能性があります。 |  
| 高S/EB案が不利になる | スチーム加熱・冷却負荷が増え、熱回収しても外部負荷が残る可能性があります。 |

書き方としては、次の主張が自然です。

\`\`\`text  
従来の手順では、反応器・分離器の操作条件を決めた後にヒートインテグレーションを行っていた。しかし、ヒートインテグレーションによるユーティリティ削減幅が大きい場合、  
操作条件の最適点自体が変わる可能性がある。そのため本設計では、HIなし評価とHIあり評価を比較し、熱回収を考慮した場合に反応器最適条件が変化するかを検討する。  
\`\`\`

これは客観的に見てもよいテーマです。過去レポート7への批判としても妥当ですが、書き方は「過去レポの誤り」ではなく、「後段HIによる改善幅が大きいため、HIを含めた再評価の余地がある」とするのが適切です。

### 7.1 TQ線図

なるべく早めに TQ 線図を書いて, 適切にプロセス内で熱交換が行われているかを見る。  
与熱流体と受熱流体の TQ 線図(コンポジットカーブ)が大きく離れていれば, 大量のスチー  
ムや冷媒が必要になってしまうので早めに改善してコンポジットカーブどうしを近づける  
必要がある。例えば, 蒸留塔の還流比を下げる(&段数を増やす)とリボイラーとコンデンサ  
ーの熱量を減らすことができる。また, 蒸留塔の圧力は塔内温度と正の相関がある(圧力を  
かけると温度を上げないと沸騰しづらくなるから)ことを考慮して, 蒸留塔の圧力を変更す  
ることで与熱流体と受熱流体のコンポジットカーブを近づけることができるかもしれない。
